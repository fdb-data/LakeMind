## 二、实施步骤（按执行顺序）

### 步骤 1：清理所有容器和镜像

```powershell
# 停止并删除所有 LakeMind 容器
docker compose --env-file LakeMindServer/.env --profile ray --profile all down
docker rm -f $(docker ps -aq --filter "name=lakemind-")  # 兜底

# 删除所有 LakeMind 相关镜像
docker images --format "{{.Repository}}:{{.Tag}}" | Select-String "lakemind" | ForEach-Object { docker rmi -f $_ }
# 也删除 meeting-agent 镜像
docker rmi -f meeting-agent-meeting-agent 2>$null

# 清理悬空镜像
docker image prune -f
```

### 步骤 4：生成 Python 锁文件（uv.lock）

在以下 4 个目录生成 `uv.lock`：

### 步骤 5：合并 3 MCP 为 mcp-suite 镜像

**新建 `LakeMindMCP/pyproject.toml`**:
```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "lakemind-mcp-suite"
version = "0.2.0"
requires-python = ">=3.11"
dependencies = [
    "mcp>=1.28,<2",
    "pydantic>=2.7",
    "pydantic-settings>=2.3",
    "pyyaml>=6.0",
    "structlog>=24.1",
    "httpx>=0.27",
]

[tool.hatch.build.targets.wheel]
packages = [
    "LakeMindAssetMCP/src/lakemind_asset_mcp",
    "LakeMindDataMCP/src/lakemind_data_mcp",
    "LakeMindAdminMCP/src/lakemind_admin_mcp",
]
```

**新建 `LakeMindMCP/Dockerfile`**:
```dockerfile
# syntax=docker/dockerfile:1.7
ARG PYTHON_IMAGE=python:3.12-slim

FROM ${PYTHON_IMAGE} AS builder
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends build-essential && rm -rf /var/lib/apt/lists/*
COPY --from=ghcr.io/astral-sh/uv:0.11.28 /uv /uvx /bin/
ENV UV_LINK_MODE=copy UV_COMPILE_BYTECODE=1 PYTHONDONTWRITEBYTECODE=1

COPY pyproject.toml uv.lock ./
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --frozen --no-dev --no-install-project

COPY LakeMindAssetMCP/src ./LakeMindAssetMCP/src
COPY LakeMindDataMCP/src ./LakeMindDataMCP/src
COPY LakeMindAdminMCP/src ./LakeMindAdminMCP/src
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --frozen --no-dev --no-editable

FROM ${PYTHON_IMAGE} AS runtime
WORKDIR /app
ENV PATH="/app/.venv/bin:${PATH}" \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    LAKE_CONFIG=/etc/lakemind/config.yaml
COPY --from=builder /app/.venv /app/.venv
CMD ["python", "-m", "lakemind_asset_mcp"]
```

删除 3 个独立 MCP Dockerfile（`LakeMindMCP/LakeMindAssetMCP/Dockerfile` 等）。

#### 7.3 `LakeMindControlCenter/Dockerfile`（多阶段 + npm ci + uv）
```dockerfile
# syntax=docker/dockerfile:1.7
ARG NODE_IMAGE=node:20-alpine
ARG PYTHON_IMAGE=python:3.12-slim

FROM ${NODE_IMAGE} AS frontend-build
WORKDIR /build
COPY frontend/package.json frontend/package-lock.json ./
RUN --mount=type=cache,target=/root/.npm \
    npm ci
COPY frontend/ ./
RUN npm run build

FROM ${PYTHON_IMAGE} AS py-builder
WORKDIR /app
COPY --from=ghcr.io/astral-sh/uv:0.11.28 /uv /uvx /bin/
ENV UV_LINK_MODE=copy UV_COMPILE_BYTECODE=1 PYTHONDONTWRITEBYTECODE=1
COPY control/pyproject.toml control/uv.lock ./
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --frozen --no-dev --no-install-project

FROM ${PYTHON_IMAGE} AS runtime
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends nginx supervisor && rm -rf /var/lib/apt/lists/*
ENV PATH="/app/.venv/bin:${PATH}" \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1
COPY --from=py-builder /app/.venv /app/.venv
COPY bff/app.py ./bff/app.py
COPY steward/app.py ./steward/app.py
COPY steward/steward_service.py ./steward/steward_service.py
COPY --from=frontend-build /build/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
COPY supervisord.conf /etc/supervisor/conf.d/lakemind.conf
RUN rm -f /etc/nginx/sites-enabled/default
EXPOSE 3000
CMD ["supervisord", "-n", "-c", "/etc/supervisor/conf.d/lakemind.conf"]
```

#### 7.4 `LakeMindServer/docker/postgres-age/Dockerfile`（多阶段，去掉 build-essential）

## 三、执行顺序

```
1. 清理容器和镜像
2. 删除旧文件
3. 创建 .dockerignore × 4
4. 创建 MCP 统一 pyproject.toml + 删除 3 个独立 Dockerfile
5. 创建 ControlCenter control/pyproject.toml
6. 生成 uv.lock × 4（uv lock）
7. 生成 package-lock.json（npm install）
8. 重写所有 Dockerfile × 5（Server, ModelServing, MCP, ControlCenter, postgres-age）
9. 创建 docker-bake.hcl
10. 重写 docker-compose.yml + 新建 docker-compose.build.yml + .env.example
11. 更新 GitHub Actions
12. 更新 AGENTS.md
13. 拉取官方 Ray 镜像
14. Bake 构建全部 5 个核心镜像
15. 启动全部容器
16. 运行 verify_full.py 全面测试
```

## 四、验收标准

1. `docker compose config --images` 不出现 `lakemind/*` 短镜像名（除 dev 覆盖）
2. 默认 docker-compose.yml 不包含 `build` 和 `pull_policy: never`
3. 5 个自研镜像：postgres-age, server-api, mcp-suite, model-serving, control-center
4. 3 个 MCP 使用同一 mcp-suite 镜像 digest
5. Ray 使用官方镜像
6. ModelServing 不再继承 server-api
7. 所有 Python 镜像使用 uv.lock 冻结安装
8. ControlCenter 使用 package-lock.json + npm ci
9. Server 构建上下文不包含 docker/postgres-age
10. verify_full.py 全部 PASS
