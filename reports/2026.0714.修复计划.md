## 2. 修复方案

### Phase 1：P1 缺陷修复（阻断发布，必须完成）

#### 1.1 DEF-001：实现 BFF 登录端点

**根因**：
- `principals` 表无 `password_hash` 列
- `AuthorizationService` 无 `login` 方法
- `security.py` 无 `/auth/login` 端点
- `/api/v1/security/auth/login` 不在 `PUBLIC_PATHS` 中
- 无 seed admin 用户

**修复步骤**：

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | 新建 `migrations/versions/007_auth_credentials.py` | ALTER TABLE principals ADD COLUMN username TEXT UNIQUE; ADD COLUMN password_hash TEXT; |
| 2 | `security/middleware.py` | `PUBLIC_PATHS` 添加 `/api/v1/security/auth/login` |
| 3 | `services/authorization_service.py` | 添加 `login(username, password_hash)` 方法：查询 principals → 验证 password_hash → 调用 `issue_token` → 返回 `{token, role, tenant_id}` |
| 4 | `api/security.py` | 添加 `POST /auth/login` 端点 |
| 5 | `007_auth_credentials.py` | seed admin 用户：username=`admin`, password_hash=sha256(`lakemind-admin-2026`), tenant_id=`default`, role=`platform_admin` |

**验证**：
```bash
# 登录
curl -X POST http://localhost:3001/auth/login -d '{"username":"admin","password":"lakemind-admin-2026"}'
# 应返回 {session_id, role}
# 访问 BFF 端点
curl http://localhost:3001/overview -H "Cookie: session_id=..."
# 应返回 200
```

**预估工作量**：0.5 天

---

#### 1.2 DEF-002：添加 v2 Secret REST 端点

**根因**：`SecretService` 有完整实现（create/resolve/rotate/list），但无 REST 端点暴露。旧 `/api/v1/metadata/secrets` 用 `X-Tenant-Id` Header 取 tenant，有跨租户风险。

**修复步骤**：

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | 新建 `api/secrets_v2.py` | 5 个端点：POST `/secrets`、GET `/secrets`、GET `/secrets/{ref}`、POST `/secrets/{ref}/rotate`、DELETE `/secrets/{ref}` |
| 2 | `api/secrets_v2.py` | 所有端点用 `get_security_context(request)` 取 tenant，不用 Header |
| 3 | `app.py` | 注册 `secrets_v2.router` at `/api/v1/secrets` |
| 4 | `api/secrets.py` | 旧端点添加 deprecation header `X-Deprecated: use /api/v1/secrets` |

**验证**：
```bash
# 创建 secret
curl -X POST http://localhost:10823/api/v1/secrets -H "Authorization: Bearer <token>" -d '{"name":"test","value":"secret123","scope":"default"}'
# 应返回 {ref: "secret://default/test"}
# 跨租户测试：用 tenant A token 访问 tenant B secret → 403
```

**预估工作量**：0.5 天

---

#### 1.3 DEF-003：实现 Reconciler `scan_jobs`

**根因**：`reconciliation_service.py:48` `scan_jobs` 返回空列表。

**修复步骤**：

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | `services/reconciliation_service.py` | 实现 `scan_jobs()`：查询 `job_runs` 表，检测：① RUNNING 超过 timeout → 记录 `job_timeout_drift`；② CANCELLING 停滞 → 记录 `stale_cancelling`；③ Ray Job 消失但状态非 LOST → 记录 `orphaned_job` |
| 2 | `services/reconciliation_service.py` | `scan_all()` 添加 `scan_jobs()` 调用 |

**验证**：
```bash
# 提交一个 Job，手动改状态为 RUNNING 但不更新 timestamp
# 运行 Reconciler，应检测到 drift
curl http://localhost:10823/api/v1/operations -H "Authorization: Bearer <token>"
# 应看到 job_timeout_drift 记录
```

**预估工作量**：0.5 天

#### 2.2 DEF-008：Bootstrap 导入默认模型

**根因**：`bootstrap.py` 未导入 models.yaml；`import_from_yaml` 期望的 YAML schema（`models`/`profiles`）与实际 schema（`llm_providers`/`embedding`/`asr`）不匹配。

**修复步骤**：

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | `scripts/bootstrap.py` | 添加 `bootstrap_models()` 函数，解析 models.yaml 的 `llm_providers`/`embedding.built_in`/`asr.built_in` 三个 section |
| 2 | `bootstrap_models()` | 为每个模型调用 `ModelManagementService.create_model()` + `create_deployment()` |
| 3 | `bootstrap_models()` | 为嵌入模型创建 `EmbeddingSpace`（dim=768, metric=cosine） |
| 4 | `bootstrap_models()` | 创建默认 Profile + Route |

**导入的默认模型**：

| model_id | model_type | provider_family | embedding_dim |
|----------|------------|-----------------|---------------|
| `deepseek-v4-flash` | chat | openai | — |
| `jina-embeddings-v2-base-zh` | embedding | fastembed | 768 |
| `iic/SenseVoiceSmall` | asr | funasr | — |

**验证**：
```bash
curl http://localhost:10823/api/v1/models/definitions -H "Authorization: Bearer <token>"
# 应返回 3 个模型定义
curl http://localhost:10823/api/v1/models/deployments -H "Authorization: Bearer <token>"
# 应返回 3 个部署
```

**预估工作量**：0.5 天

#### 2.4 DEF-004 + DEF-005：架构债务清理（可分阶段）

**DEF-004 旧路由清理**：

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | `app.py` | 评估 12 个旧路由（objects/tables/vectors/kv/graph/sql/jobs/memory/metadata/secrets/embedding/llm） |

**预估工作量**：v0.2.0 标记 deprecated 0.5 天；完全迁移 3+ 天（建议 v0.3.0）

## 7. 风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| 重建 model-serving 镜像可能耗时较长（funasr + torch 依赖大） | 可先验证 Dockerfile 正确性，后台构建 |
| 移除 PG 端口映射影响本地调试 | 保留 `docker exec` 方式或临时映射 |
| 修改默认 API Key 需同步更新所有引用 | 全局搜索替换，验证脚本同步更新 |
| Bootstrap 模型导入需在迁移后执行 | `migrate` 容器完成后 → `bootstrap` 容器 → `server-api` |
| 旧路由标记 deprecated 可能影响 MCP 兼容性 | 仅添加 header，不改变行为 |
