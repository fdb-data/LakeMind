## LakeMind 产品概要（面向开发团队）

### 1. 产品愿景
**LakeMind 是一个面向 AI Agent 的多模智能数据湖产品**，为企业内成百上千个 Agent 提供统一、安全、可治理的结构化数据、知识库（向量/文档）、短期/长期记忆和 Skills 的读写能力。  
核心理念：**Agent 是一等公民，人类通过观察与委托进行管理**，所有访问通过标准 MCP 协议收敛。

### 2. 系统分层总览
整个 LakeMind 分为两大平面：

**（1）数据平面（LakeMind Server）**  
- **存储底座**：SeaweedFS（统一文件/对象存储，S3 兼容）  
- **存储引擎**：Iceberg（结构化）、Lance（多模态/向量）、Dragonfly（缓存/短期记忆）  
- **计算引擎**：Daft（DataFrame，MVP 单机）  
- **元数据层**：Apache Gravitino（统一编目，管理表与文件集）  
- **安全治理（生产）**：Apache Ranger  

**（2）管理平面（三件套）**  
- **LakeMindMCP**：MCP 协议网关，Agent 的唯一入口  
- **LakeMindSteward**：运维管理智能体，接收人类自然语言指令，调用 MCP 管理工具  
- **LakeMindMonitor**：只读监控仪表板 + 与 Steward 对话的聊天界面

### 3. 数据域与存储映射
| 数据域 | 落地引擎 | 说明 |
|--------|----------|------|
| 结构化数据 | Iceberg 表，Gravitino 编目 | Agent 通过 SQL 读写 |
| 知识（多模态 RAG） | Lance 存储 + LanceDB 检索，Gravitino Fileset 管理 | 向量/文本/图片的统一检索 |
| 短期/工作记忆 | Dragonfly，带 TTL 的 KV | 对话上下文、临时状态 |
| 长期/语义记忆 | Lance 向量 + Iceberg 元信息表 | 可回溯的持久记忆 |
| Skills | 文件存 SeaweedFS，元信息存 Iceberg，向量索引存 Lance | 代码、Prompt、描述的管理与发现 |

### 4. 多租户模型（从 MVP 即内建）
- **租户** = 一个业务组（如零售、对公、风控），资源完全隔离。
- **隔离方式**：以 Gravitino Catalog 为边界，存储用 `tenant_id/` 前缀，Dragonfly 分 DB，LanceDB 分数据库。
- **Agent 凭证**：Token 中编码 `tenant_id`，MCP Server 自动注入上下文，杜绝跨租户访问。

### 5. MVP 阶段交付目标（Phase 1）
- **LakeMind Server** 已部署为单机 `docker-compose`，包含所有数据平面组件。
- **LakeMindMCP**（本次开发重点）：实现 MCP 协议、基础多租户、数据域工具（SQL/向量/记忆/Skill）、管理域工具、元数据资源发现。
- **LakeMindSteward**：具备对话式运维和自主巡检能力的管理 Agent。
- **LakeMindMonitor**：只读仪表板（数据地图、组件健康、调用量） + 内嵌 Steward 聊天窗口。

### 6. 开发顺序与依赖
1. **LakeMindMCP**：优先开发，因为它是所有 Agent 和 Steward 的基础。MVP 只需 2 个静态 Token 区分业务 Agent 和 Steward。
2. **LakeMindMonitor**：前端界面与 BFF，依赖 MCP Server 的资源和健康检查接口。
3. **LakeMindSteward**：依赖 MCP Server 的 admin 域工具和 Monitor 提供的聊天通道。

当前首要任务：**完成 LakeMindMCP 的详细开发，尽快让第一个业务 Agent 通过 MCP 成功读写数据湖**。
